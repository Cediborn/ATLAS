Philosophy

Why another system?

Most productivity tools are fast to ship and slow to trust. Atlas is designed the other way around — boring, proven technology, built to outlast the apps you'll abandon next year.

  • 01
    Calm by default

    Minimal chrome. Content over UI. No notification badges competing for attention. The interface gets out of the way.

  • 02
    Fast is a feature

    Every interaction feels instant. Optimistic UI, no spinners for common actions. If you feel a delay, something is wrong.

  • 03
    One system, not seven apps

    Tasks, notes, calendar, habits, learning, and finance share the same primitives and reference each other. Universal search spans all of it.

  • 04
    Keyboard-first, mouse-optional

    The command palette (⌘K) is the primary way to move around and act. You should rarely need to reach for the mouse.

  • 05
    You own your data

    Everything exports to JSON and Markdown. No lock-in. No dark patterns. Your data works without Atlas if you ever leave.

  • 06
    AI is ambient, not a chatbot

    The AI shows up inline — quick-capture parsing, summaries, suggestions — not a separate tab you have to remember to open.

  • 07
    Built to last

    Boring, proven technology. Premium comes from execution and taste, not trend-chasing. TypeScript, PostgreSQL, React — nothing exotic.

The eight pillars

Every area of your life,
in one coherent system.

Each pillar is a typed domain with its own schema. Together, they're held by a universal linking layer — so a task can reference a note, a habit can link to a project, and nothing lives in a silo.

Projects & Tasks
Project · Task

Hierarchical projects that link to notes, events, and each other. Every task is a first-class entity with history.

Notes
Note (block-based)

Freeform, block-based notes that can reference any task, event, or resource. Backlinks surface automatically.

Calendar
Event

Two-way link between tasks and events. Schedule a task and it becomes an event. Complete it and both update.

Habits
Habit · HabitLog

Daily and weekly cadences with streak tracking. Simple enough to log in two taps; powerful enough to show trends over years.

Learning
Resource · Progress

Track books, courses, and articles with per-resource progress. Link learnings directly to projects or notes.

Finance
Account · Transaction

Manual entry to start, CSV import to follow. Your financial picture lives alongside everything else — no separate app needed.

AI Layer
Ambient · cross-cutting

Capture parsing, summarization, weekly review generation. AI shows up where you already work — never in a separate tab.

Analytics
Activity log · cross-cutting

Derived from an append-only event log built in from day one. Dashboards that span every pillar — tasks, habits, finance, learning.

Ambient intelligence

AI that shows up
where you're already working.

No chat sidebar. No "ask Atlas" button. The AI parses what you type, summarizes what you've done, and surfaces what matters — without making itself the center of attention.

Universal linking

Everything can
reference everything.

A generic links table sits beneath all eight pillars. Any entity can point to any other — task to note, note to event, habit to project. No redesigning the data layer to add a new connection.

Backlinks surface automatically. The AI layer uses the graph to generate context-aware summaries and surface related items without you asking.

Roadmap

Phases, not features.

Atlas is built in deliberate vertical slices — thin but real, proven before the next layer goes on. No moonshot features before the foundation is solid.

Phase 0
Foundation done

Architecture decisions, token system, folder structure, naming conventions. The document you're reading is the deliverable.

Phase 1
App shell current

Auth, core layout, command palette (⌘K), design system components. The skeleton every future pillar hangs on.

Phase 2
Tasks + Notes next

First two pillars end-to-end with cross-linking. Proves the architecture before five more modules go on top of it.

Phase 3
Calendar + Habits

Two more pillars. The activity event log goes live here — cheap now, expensive to backfill later.

Phase 4
Learning + Finance

The remaining pillars. By now every domain exists and the linking graph is meaningful.

Phase 5
AI layer

Quick-capture NLP parsing, inline summaries, weekly review generation. AI built on top of real data — not bolted on day one.

Phase 6–9
Analytics → Desktop → Mobile → Extensions

Cross-pillar dashboards, Tauri desktop shell, React Native, plugin API. Each phase is optional until it hurts not to have it.
